
# Theme System Implementation Plan

## Overview
Add a 4-mode theme system (System Default, Dark, Light, Blossom Pink) with DataStore persistence and instant app-wide switching. Replace hardcoded colors in composables with MaterialTheme.colorScheme references.

---

## Step 1: Define ThemeMode enum
**New file: `data/ThemeMode.kt`**
- Create `enum class ThemeMode { SYSTEM_DEFAULT, DARK, LIGHT, BLOSSOM_PINK }`
- Follows the same pattern as `AlbumSortOption` / `PhotoSortOption`

## Step 2: Add THEME_MODE key to DataStore
**Modify: `data/SettingsKeys.kt`**
- Add `val THEME_MODE = intPreferencesKey("theme_mode")`
- Add `val THEME_MODE = ThemeMode.SYSTEM_DEFAULT.ordinal` to `Defaults`

## Step 3: Add theme flow + setter to SettingsRepository
**Modify: `data/SettingsRepository.kt`**
- Add `themeMode: Flow<ThemeMode>` (same IOException-safe pattern as existing flows)
- Add `suspend fun setThemeMode(mode: ThemeMode)`
- Also expose a `themeModeFlow` that can be collected independently of the ViewModel (since theme state must be hoisted to `MainActivity` level)

## Step 4: Define 4 complete ColorSchemes
**Modify: `ui/theme/Color.kt`**
- Keep existing Gemini dark palette constants
- Add Light palette constants (white surfaces, Gemini Blue accents)
- Add Blossom Pink palette constants (soft pinks, warm grays)

**Modify: `ui/theme/Theme.kt`**
- Replace the single `DarkColorScheme` and fake `LightColorScheme` with 4 proper schemes:
  - `DarkColorScheme` — current Gemini dark (unchanged)
  - `LightColorScheme` — white bg, Gemini Blue accents
  - `BlossomPinkColorScheme` — light theme with pink primary/secondary
- Remove the dynamic color override path (it overwrites everything with hardcoded dark values anyway)
- Add `blossomPinkColorScheme()` function
- Update `ProxaGalleryTheme` to accept `ThemeMode` instead of `darkTheme: Boolean`
- Theme selection logic: if `ThemeMode.SYSTEM_DEFAULT`, check `isSystemInDarkTheme()` and pick dark/light; otherwise use the chosen scheme directly
- Each scheme sets ripple to match its primary color

## Step 5: Wire theme state into MainActivity
**Modify: `MainActivity.kt`**
- Create `SettingsRepository` directly in `MainActivity` (not through ViewModel) since theme wraps the entire compose tree
- Collect `themeMode` flow with `collectAsStateWithLifecycle`
- Pass resolved theme to `ProxaGalleryTheme`

## Step 6: Add Theme Picker Dialog in SettingsTab
**Modify: `ui/HomeScreen.kt`** — SettingsTab section only
- Add `ThemePickerDialog` composable (Material 3 AlertDialog with radio buttons)
- Add `showThemeDialog` state to `SettingsTab`
- Make the Theme `SettingsItem` clickable → opens dialog
- Dialog options: System Default, Dark, Light, Blossom Pink with radio buttons
- On selection: call ViewModel's `setThemeMode()` and dismiss
- Update Theme subtitle to show current theme name

## Step 7: Wire setThemeMode into GalleryViewModel
**Modify: `ui/GalleryViewModel.kt`**
- Add `setThemeMode(ThemeMode)` method that delegates to `SettingsRepository`

## Step 8: Replace hardcoded colors with MaterialTheme.colorScheme
**Modify: `ui/HomeScreen.kt`** — color replacements only, no layout/logic changes
**Modify: `ui/PhotoViewerScreen.kt`** — color replacements only
**Modify: `ui/RecentlyDeletedScreen.kt`** — color replacements only
**Modify: `ui/MediaDetailsSheet.kt`** — color replacements only

Replacement strategy (mapping table):

| Hardcoded Color | Replacement |
|---|---|
| `Color(0xFF090B10)` backgrounds | `MaterialTheme.colorScheme.background` |
| `Color(0xFF10131A)` / `Color(0xFF161A22)` surfaces | `MaterialTheme.colorScheme.surface` or `surfaceContainerHigh` |
| `Color(0xFF131722)` / `Color(0xFF1E2330)` cards | `MaterialTheme.colorScheme.surfaceVariant` |
| `Color.White` (text/icons) | `MaterialTheme.colorScheme.onSurface` or `onBackground` |
| `Color.White.copy(alpha = 0.6f)` (subtitle text) | `MaterialTheme.colorScheme.onSurfaceVariant` |
| `Color.White.copy(alpha = 0.08f)` (dividers) | `MaterialTheme.colorScheme.outlineVariant` |
| `Color.White.copy(alpha = ~0.2-0.3f)` (borders) | `MaterialTheme.colorScheme.outline` |
| `Color(0x1AFFFFFF)` (glass borders) | `MaterialTheme.colorScheme.outline` |
| `Color(0x1AFFFFFF)` (icon backgrounds) | `MaterialTheme.colorScheme.surfaceVariant.copy(alpha=0.5f)` |
| `Color(0xFF6EA8FE)` (accent) | `MaterialTheme.colorScheme.primary` |
| `Color(0xFF9B7BFF)` (purple accent) | `MaterialTheme.colorScheme.secondary` |
| `Color(0xFF5EE6FF)` (cyan accent) | `MaterialTheme.colorScheme.tertiary` |
| `Color(0xFFEFB8C8)` (pink accent) | Keep as-is (gradient decoration, not a semantic slot) |
| `Color(0xFFFF5252)` / `Color(0xFFFF1744)` (delete/error) | `MaterialTheme.colorScheme.error` |
| `Color.Black.copy(alpha=...)` (overlays/scrim) | `Color.Black.copy(alpha=...)` (theme-neutral, keep) |

For the glass-morphism effects (semi-transparent surfaces), the dark theme will use dark-based semi-transparent colors and the light themes will use white-based semi-transparent colors. This is handled by deriving from `MaterialTheme.colorScheme.surface` with alpha rather than hardcoded hex values.
