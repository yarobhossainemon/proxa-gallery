# Proxa Gallery — Navigation Redesign (Final Plan)

## New 5-tab structure
| Idx | Tab     | Icon                          |
|-----|---------|-------------------------------|
| 0   | Photos  | `Icons.Rounded.Photo`         |
| 1   | Albums  | `Icons.Rounded.Folder`        |
| 2   | AI      | `Icons.Rounded.AutoAwesome`   |
| 3   | Library | `Icons.Rounded.PhotoLibrary`  |
| 4   | Settings| `Icons.Rounded.Settings`      |

## Migration: `selectedTab` re-indexing
`selectedTab: Int` is persisted in DataStore. **In `observeSettings`, clamp persisted `tabIndex` to `0..4`**; out-of-range values (from the old 5-tab scheme) become `0` and are re-persisted. No schema change.

VM branching changes in `prepareViewer` / `selectAllOrNone`: remove old `currentTab == 1` (Search) and `currentTab == 3` (Favorites) branches. Search now lives inside Photos (tab 0): if `searchQuery` non-blank → search IDs, else all IDs. Favorites lives behind a route, resolved by new `prepareFavoritesViewer()`.

## Change 1 — Reusable grid component: `MediaGrid`

**New file `MediaGrid.kt`** — a reusable **grid component** (NOT a screen). It renders: an optional title bar (title + back + sort menu), drag-selection, selection top/bottom bars, empty state, and the existing `PhotoGridItem`. Screens host it.

```kotlin
@Composable
fun MediaGrid(
    title: String,
    photos: List<MediaItem>,
    uiState: GalleryUiState,
    onBackClick: () -> Unit,
    onPhotoClick: (MediaItem) -> Unit,
    onToggleFavorite: (Long, Boolean) -> Unit,
    onEnterSelectionMode: (Long) -> Unit,
    onTogglePhotoSelection: (Long) -> Unit,
    onSelectPhoto: (Long) -> Unit,
    onExitSelectionMode: () -> Unit,
    onSelectAllVisible: () -> Unit,
    onShareSelected: () -> Unit,
    onFavoriteSelected: () -> Unit,
    onRequestBulkDelete: () -> Unit,
    onConfirmBulkDelete: () -> Unit,
    onCancelBulkDelete: () -> Unit,
    showDeleteConfirmation: Boolean,
    onPhotoSortChange: (PhotoSortOption) -> Unit,
    emptyIcon: ImageVector,
    emptyTitle: String,
    emptySubtitle: String,
    showSortMenu: Boolean = true
)
```

Reuses (lifted `private` → `internal` in `HomeScreen.kt`, **no behavior change**): `PhotoGridItem`, `SelectionBottomBar`, `SelectionTopBar`, `TopBarSlot`, `SelectionCheckmark`, `PhotoSortMenuButton`, `GeminiEmptyState`, `GeminiLoadingRing`, `allSelectedAreFavorite`.

**Scope discipline:** `MediaGrid` is the reusable home for *every photo-grid destination* the user listed: **Favorites, AI Collections, future Videos, Screenshots, Documents, Downloads, Similar Photos, Smart Albums.** Not forced onto Hidden Albums / Recently Deleted (they render `Album`/`TrashItem`, have domain-specific actions, and use `ViewerMode.TRASH`).

**Used immediately by:** `FavoritesScreen`, `VideosScreen`, `ScreenshotsScreen`, `DocumentsScreen`, etc. when they're added — each is a thin wrapper that supplies its own `photos` list and route.

## Change 3 — Library: generic `List<LibrarySection>`, not three hardcoded cards

**`LibrarySection` is a generic list from day one.** Each item is one object — adding Videos/Documents/Screenshots/Downloads/GIFs/RAW/Motion/Burst later means appending one `LibrarySection`, never rewriting the UI.

```kotlin
data class LibrarySection(
    val title: String,
    val subtitle: String,
    val icon: ImageVector,
    val gradient: Pair<Color, Color>,
    val count: Int? = null,
    val onClick: () -> Unit,
    val enabled: Boolean = true
)
```

**`LibraryTab()` builds `List<LibrarySection>`** and renders it via `LazyColumn { items(sections) { LibrarySectionCard(it) } }`:

**Initially enabled (3):**
- ❤️ Favorites → `onFavoritesClick` (new route)
- 🙈 Hidden Albums → existing `onHiddenAlbumsClick` (count = `hiddenAlbums.size`)
- 🗑 Recently Deleted → existing `onRecentlyDeletedClick`

**Future-ready (added later as one object each, NOT built now):** Videos, Screenshots, Documents, Downloads, GIFs, RAW, Motion Photos, Burst Photos, AI Collections, People, Places, Storage Analysis. Disabled ones render greyed + non-clickable when added.

`LibrarySectionCard(section)` — large rounded card, glassmorphism, icon in gradient circle, title + subtitle + optional count badge, premium press animation matching `AlbumCard`.
