## Library Tab → Premium Dashboard (UI only)

**Scope:** Single file — `app/src/main/java/com/emon/proxagallery/ui/HomeScreen.kt`. No ViewModel, Repository, Navigation, DB, AI, or MediaStore changes. All required Glass components (`GlassCard`, `GlassSurface`, `GlassBadge`, `GlassCountBadge`, `cardShape`, `GlassTokens`) and icons are already imported.

- **No album list below** (Albums has its own tab; user opted to keep as-is).

### Concrete changes in `HomeScreen.kt`

#### 1. Call site (lines ~818–826) — pass real counts
`LibraryTab(...)` gains three params wired from `uiState` (all already available, no ViewModel change):
- `albumsCount = uiState.albums.size`
- `favoritesCount = uiState.favoriteItems.size`
- `recentlyDeletedCount = 0`  *(TrashViewModel is separate and not in scope; pass 0 → badge hidden. Kept as a parameter so wiring it later is a one-line change.)*

`hiddenAlbumsCount` stays as-is.

#### 2. `LibraryTab` signature (lines 2935–2940) — extend params
Add `albumsCount: Int`, `favoritesCount: Int`, `recentlyDeletedCount: Int` to the existing `hiddenAlbumsCount`.

#### 3. Replace the bare `Column` body (lines 2974–3006) with the new structure
- Keep: scrollable `Column`, status-bars padding, `"Library"` title (28.sp Bold), `"Your content, organized"` subtitle (14.sp).
- **Add section label** `"DASHBOARD"` (small uppercase, `onSurfaceVariant`, letter-spacing) + the **stat grid GlassCard** (new private composable, see below).
- **Add section label** `"QUICK ACCESS"` + the existing three `LibrarySectionCard` rows — but now Favorites and Recently Deleted also pass `count` so **all three render a `GlassCountBadge`** uniformly (currently only Hidden Albums does).
- Keep bottom 100.dp nav-bar spacer.

#### 4. New private composable `LibraryStatCard(...)` (~40 lines)
A single `GlassCard(shape = cardShape(), elevation = 0.dp)` containing:
- A small `Row` section label (`"DASHBOARD"`).
- A 2×2 grid built from two `Row`s (each `weight(1f)`), separated by a thin `HorizontalDivider` — reuses the existing imported divider. Each of the 4 cells = `LibraryStatTile(...)`.
- Values computed **once** via `remember(albumsCount) { ... }` outside the tiles to avoid recomposition cost.

#### 5. New private composable `LibraryStatTile(icon, label, value, modifier)` (~30 lines)
Each tile: 48.dp circular gradient icon box (same style as `LibrarySectionCard`), a small label (12.sp, `onSurfaceVariant`), and a bold value (20.sp, `onSurface`). For Photos/Videos/Storage the value is the em-dash string `"—"`; Albums uses the real integer formatted with grouping.

#### 6. `LibrarySection` list (lines 2944–2970) — add counts to all three
- Favorites: add `count = favoritesCount`
- Recently Deleted: add `count = recentlyDeletedCount`
- Hidden Albums: unchanged.
`LibrarySectionCard` already renders `GlassCountBadge` whenever `count != null && count > 0`, so this single change makes badges uniform. No change to `LibrarySectionCard` itself.

### What stays untouched
- `LibrarySection` data class, `LibrarySectionCard` composable, click handlers, navigation chain (`GalleryNavHost`), `GalleryViewModel`, `GalleryRepository`, `TrashViewModel`, all DB/AI/MediaStore logic.
- Existing icon/gradient choices per card (Favorites forward gradient, Hidden/Trash reversed gradient).

### Performance
- Dashboard values computed once via `remember(counts)` — no per-tile recomputation.
- Reuses existing `uiState` fields; no new state, no new collectors.
- Stat tiles are plain `GlassCard`+`Row` with no animation/`pointerInput`, so they add negligible recomposition cost.
