## Fix: Align favorites sort semantics with paged grids

**Root cause:** `sortFavoriteItems()` sorts by `dateTakenMs` (DATE_TAKEN, or DATE_ADDED fallback), while the paged grids sort by MediaStore's `DATE_ADDED` column. "Newest" means different things in different tabs.

**Approach:** Add a `dateAddedMs` field to `MediaItem` and use it in `sortFavoriteItems()` for NEWEST/OLDEST, matching the paged grids exactly.

### Changes (2 files)

**1. `MediaItem.kt`** — Add `dateAddedMs` field
- Add `val dateAddedMs: Long? = null` to the data class constructor (after `dateTakenMs`)
- No default value change needed — it's nullable with `null` default

**2. `GalleryRepository.kt`** — Populate the new field
- In `queryMediaItems()`, store the DATE_ADDED value (already read as `dateAdded = cursor.getLong(dateAddedColumn) * 1000L`) into the `MediaItem` constructor as `dateAddedMs = dateAdded`

**3. `GalleryViewModel.kt`** — Fix the comparator
- In `sortFavoriteItems()`, change NEWEST/OLDEST from `it.dateTakenMs` to `it.dateAddedMs`
- Update the KDoc comment to remove the "deliberately out of scope" note — the semantics now match

### What this does NOT change
- Paged grid sorting (MediaStore queries) — untouched
- Album sorting — untouched
- `dateTakenMs` field — remains as-is (used by viewer/details)
- Viewer order for favorites — will now match the visible grid order
