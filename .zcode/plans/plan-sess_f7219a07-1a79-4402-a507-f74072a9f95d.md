## Signature Bottom Navigation — Implementation Plan

### Goal
Build `SignatureGlassNav.kt` — the app's visual-identity navigation. A single continuous glass dock whose top edge rises in a smooth convex "molten hill" that the AI button grows out of (not floats above). Delivered as a self-contained component with Compose Previews in all three themes (Dark / Light / Blossom). Not wired into `HomeScreen` in this pass — visual review first.

### Design system: reuse, don't duplicate
Every primitive already exists in `ui/glass/`. The new file imports and composes them — no new dependencies, no token duplication:
- `Modifier.glassBackground(shape, strong)` — frosted fill + hairline border + top highlight
- `Modifier.glow(shape, radius)` — ambient halo (used on the AI button only)
- `Modifier.animatedGradientBorder(shape, active)` — blue→purple→cyan animated border
- `Modifier.pressScale(pressed, spec)` — tactile feedback
- `GlassMotion.GlowPulse` — slow breathing pulse for the AI orb
- `MaterialTheme.extendedColors` — gradient stops, glowColor, innerHighlight, onNavUnselected, glassSurfaceStrong (already populated per-theme)

### The molten-glass dock (the hard part, done right)
The signature requirement: **no notch, no hole, no FAB**. Solved with one hand-drawn `Path` + a custom `Shape`:

`SignatureDockShape(hillCenterX, hillWidth, hillHeight)` implements `Shape.createOutline`:
1. Starts bottom-left, goes up the left edge, across the top-left to the base of the hill.
2. Traces the hill with **two cubic Bézier curves** forming a gentle convex bell (flat tangent at the base, flat tangent at the apex) — explicitly *not* a semicircle. The control points are tuned so the curve hugs ~70% of the AI button's circumference.
3. Continues across the top-right, down the right edge, back to start.

The hill parameters are derived from the AI button diameter (74–78dp, token = `AiButtonDiameter = 76.dp`) so they always agree. `clip(shape)` then lets the glass fill perfectly follow the silhouette.

### File: `app/src/main/java/com/emon/proxagallery/ui/glass/SignatureGlassNav.kt`

**1. Tokens block** (top of file, single source of truth — matches `GlassTokens` convention):
```
AiButtonDiameter = 76.dp
AiButtonRiseAboveDock = ~30% of diameter
DockHeight = 64.dp (excludes the hill)
DockHorizontalPadding = 16.dp
DockBottomGutter = 12.dp (clears gesture nav)
NavIconSize = 24.dp, NavLabelSp = 11.sp
HillBlendRadius = AiButtonDiameter * 0.46  // how tightly dock hugs button
```

**2. `SignatureDockShape`** — the `Shape` described above.

**3. `ProxaSymbol` composable** — an original, restrained `ImageVector` built in code (aperture + soft spark hybrid). White tint, scales crisply. Not the word "AI", not Material's AutoAwesome.

**4. `AiGlassButton` composable** — the hero:
- `glow(circleShape, radius = 22.dp)` ambient cyan halo underneath (restrained — `colors.glowColor` is already ~0.20 alpha dark).
- Breathing animation: `rememberInfiniteTransition` + `GlowPulse` driving a 1.0→1.04 scale and a subtle glow-alpha breathe (2.2s, reverse — exactly the existing `GlowPulse` spec).
- Frosted-glass center via `glassBackground(CircleShape, strong = true)`.
- Animated gradient ring (`animatedGradientBorder(CircleShape, active = true)`) — the blue→cyan→purple identity, drawn as a thin 1.5dp ring (slightly thicker than the hairline so it reads as the hero, but not neon).
- Thin white top highlight (already in `glassBackground`).
- `pressScale` on tap.
- `ProxaSymbol` centered, white.

**5. `SignatureNavItem` composable** — Samsung One UI outline icon + always-visible label:
- Icons: `Icons.Rounded.Photo` (Gallery), `Folder` (Albums), `CollectionsBookmark` (Library), `Settings` — matching the existing glass icon set.
- Active: blue accent (`colors.gradientStart`), `NavSpring` scale pop, no glow (glow is reserved for the AI button).
- Inactive: `colors.onNavUnselected`.
- Labels in `11.sp` medium, always visible per brief.

**6. `SignatureGlassNav` composable** — assembles the four items + AI centerpiece:
```
@Composable
fun SignatureGlassNav(
    selectedTab: Int,
    onTabSelected: (Int) -> Unit,
    onAiClick: () -> Unit,
    modifier: Modifier = Modifier
)
```
- `Box` sized to dock height + hill rise + bottom gutter.
- `Layout` (custom measure) places: dock behind, AI button offset so ~30% sits above the hill apex and ~70% is visually embedded.
- A `Row` with two groups: `[Gallery, Albums]` left, gap for the hill, `[Library, Settings]` right — `Arrangement.spacedBy` keeps them clear of the AI button.
- `glassBackground(SignatureDockShape(...), strong = true)` for the one continuous surface.

### Three-theme tuning (per user answer)
The same `ExtendedColors` tokens already differ per theme. I will additionally tune per-theme via a small local when-block inside the component for the two values that need bespoke treatment:
- **Dark**: glass `strong` (≈0.8 alpha), glow 0.20, ring 1.5dp, hill curve hugs tightly. The flagship look.
- **Light**: glass slightly stronger (0.95) so the silhouette stays readable on bright bg, glow reduced to existing `LightGlowColor` (0.12), ring kept vivid so the identity survives, top white highlight already brighter.
- **Blossom**: uses `BlossomGlassSurfaceStrong` + pink gradient stops automatically; ring tints pink. Glow at 0.14.

### What this plan does NOT do (per AGENTS.md "one feature at a time")
- Does not modify `HomeScreen.kt`, `GalleryNavHost.kt`, or existing `GlassNavigationBar.kt` — those are a separate integration task after you approve the visual.
- Does not add Gradley dependencies — everything uses existing Compose/Material3/glass APIs.
- Does not introduce a new destination/route.

### Verification
- `./gradlew :app:compileDebugKotlin` (or the project's existing build command) — confirms it compiles against the existing BOM and KSP setup.

- Manual review against each brief constraint (no notch / 30-70 embedding / single continuous surface / restrained glow / labels visible / 4 items / AI not a tab).

### Risk notes
- The custom `Shape.createOutline` must read `size` correctly and handle layout direction — I'll mirror the pattern already proven in `GlassModifiers.glassBackground` (which calls `shape.createOutline(size, layoutDirection, this)`).
- Hill Bézier control points are the visual crux; I'll expose them as token fractions so they're tunable post-review without touching geometry code.
