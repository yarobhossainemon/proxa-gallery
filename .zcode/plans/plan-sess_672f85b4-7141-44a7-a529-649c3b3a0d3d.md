# Proxa Gallery — Glass Design System (Phase 1) — FINAL

## Goal
One reusable glass design system shared by every main surface. Phase 1 covers **Photos, Albums, AI, Library, Settings, bottom nav, search bar, chips, cards, badges**. Identity: dark-navy base, blue→purple gradient accents, cyan glow used *sparingly*, soft white highlights, frosted surfaces, **unified spring motion** — consistent across all 3 themes (Dark / Light / Blossom Pink).

## OUT OF SCOPE (Phase 2 — not touched)
`PhotoViewerScreen.kt`, `MediaDetailsSheet.kt`, `EditAlbumBottomSheet`, `MoreBottomSheet`, all bottom sheets, `ThemePickerDialog`, all dialogs. `FavoritesScreen` / `HiddenAlbumsScreen` / `RecentlyDeletedScreen` inherit the photo-card migration for free (they reuse `PhotoGridTab`) but their own chrome is Phase 2. `viewerGlassColor()` / `viewerGlassBorder()` helpers stay as-is.

## Step 2 — `ui/glass/` package (split by purpose)

Nine focused files. Each holds one responsibility so the package scales to future components (`GlassDialog`, `GlassBottomSheet`, `GlassFAB`, `GlassSwitch`, …) without any file ballooning into another `HomeScreen.kt`.

### `GlassTokens.kt` (new)
```kotlin
object GlassTokens {
    val CardRadius = 22.dp
    val SmallRadius = 16.dp
    val ChipRadius = 50.dp
    val NavRadius = 34.dp
    val SearchRadius = 32.dp
    val GlassPadding = 14.dp
    val BorderWidth = 1.dp
    val PressScale = 0.96f
}
```
Single source of truth — no scattered `18.dp`/`20.dp`/`22.dp` across the project.

### `GlassMotion.kt` (new) — unified motion language
```kotlin
object GlassMotion {
    val PressSpring  = spring(dampingRatio = MediumBouncy,  stiffness = MediumLow)
    val CardSpring   = spring(dampingRatio = LowBouncy,     stiffness = MediumLow)
    val ChipSpring   = spring(dampingRatio = NoBouncy,      stiffness = Medium)
    val NavSpring    = spring(dampingRatio = LowBouncy,     stiffness = MediumLow)
    val FadeIn       = tween<Float>(220, easing = FastOutSlowInEasing)
    val GlowPulse    = infiniteRepeatable(tween(2200, easing = LinearEasing), RepeatMode.Reverse)
}
```
**Every** `Glass*` component reads its motion from here — this is what makes the system feel cohesive and premium (Samsung/Pixel/Apple all share this trait). One file to tune the whole app's feel.

### `GlassModifiers.kt` (new)
- `Modifier.glassBackground(shape, strong = false)` — frosted surface + 1px border + inner-highlight top stroke. **No glow.** Base look every glass surface shares.
- `Modifier.glow(shape, color, radius)` — ambient glow via `shadow(spotColor=...)`. Opt-in only.
- `Modifier.pressScale(pressed, spec = GlassMotion.PressSpring)` — spring `graphicsLayer` scale.
- `Modifier.animatedGradientBorder(shape, active)` — moving 3-stop gradient border.

### `GlassSurface.kt` (new)
- `GlassSurface` — `glassBackground` + content slot. Base container; no glow.

### `GlassCards.kt` (new)
- `GlassCard` — `GlassSurface` + small elevation shadow + optional `pressScale` slot (`GlassMotion.CardSpring`). **No glow.**
- Used by: photo card, album footer, library card, AI feature card, settings card.

### `GlassButtons.kt` (new)
- `GlassButton` — glass pill + label; gradient fill when `selected`.
- `GlassIconButton` — circular glass button with press-scale; **the only button that may glow** (AI sparkle).

### `GlassNavigation.kt` (new)
- `GlassNavigationBar` — extracted from `FloatingBottomNavigation`. Selected item = gradient circle + glow ring (one of the few glow spots) + `GlassMotion.NavSpring`. Public signature preserved.

### `GlassSearch.kt` (new)
- `GlassSearchBar` — refactored to use `Modifier.glassBackground` + `animatedGradientBorder` when focused (glow spot). `aiActionButton` slot = glowing `GlassIconButton` with sparkle. Public signature preserved.

### `GlassChips.kt` (new)
- `GlassChip` — capsule. Active = animated gradient border + soft scale (`GlassMotion.ChipSpring`, glow spot). Replaces **both** `QuickSearchChips` items and `AiExampleChips` items → dedup.

### `GlassBadge.kt` (new)
- `GlassBadge` — small glass circle; optional gradient fill for emphasis (pin / favorite / count).

## Step 3 — Migrate `HomeScreen.kt` to the system

In-place refactors, **no behavior changes**, public signatures preserved:

| Existing composable | Migrates to | Glows? |
|---|---|---|
| `FloatingBottomNavigation` | `GlassNavigationBar` (thin wrapper) | selected item only |
| `GlassSearchBar` | refactored internals, same signature | focused + AI button |
| `QuickSearchChips` + `AiExampleChips` | both → `GlassChip` (dedup) | active chip only |
| `PhotoGridItem` | `GlassCard` + 18dp radius + press-scale; favorite → `GlassBadge` | no |

| `AiFeatureCard` + AI hero search box | `GlassCard` / `GlassSearchBar`; 88dp hero orb keeps gradient + ambient `Modifier.glow` halo (the "AI orb") | orb + search only |

| `SelectionBottomBar` / `SelectionTopBar` | `GlassSurface(strong=true)`; removes hardcoded `Color(0xC0090B10)` | no |

**All hardcoded glass colors removed** — `MaterialTheme.extendedColors` becomes the only source.

## Glow budget (enforced)
Glow appears **only** on: AI sparkle button · selected nav item · active/focused search field · active chip · AI hero orb. Everything else (cards, badges, settings rows, library cards, every photo, every album) gets glass + shadow + scale, **never glow**.

---

## Execution discipline
Small batches, frequent builds, preserve existing behavior while replacing only the visual layer:
1. Tokens first (`Color.kt`, `Theme.kt`) → build.
2. `GlassTokens` + `GlassMotion` + `GlassModifiers` → build (nothing uses them yet, but must compile).
3. Each `Glass*` component file → build after each.
4. Migrate `HomeScreen.kt` composable by composable → build after each cluster (nav/search → chips → cards → selection bars).
5. Final full build + read-only grep review.

## What this does NOT change
- No new Gradle dependencies (translucent-layer + highlight-stroke approach, matching existing `FloatingBottomNavigation`; true `RenderEffect` blur is API 31+ and minSdk is 29).
- No ViewModel / Repository / data-layer / navigation-graph changes.
- No public-signature changes to existing composables — internals only.
- PhotoViewer, sheets, dialogs, and secondary screens untouched (Phase 2).
