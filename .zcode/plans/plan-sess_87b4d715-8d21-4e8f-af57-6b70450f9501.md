## Goal
Unify the Recently Deleted viewer onto the main `PhotoViewerScreen` engine (no duplicated viewer logic), parameterized by a new `ViewerMode`. Gallery stays unchanged; Trash gets Restore + Delete Forever only.

## New file: `ui/ViewerModels.kt`
- `enum class ViewerMode { GALLERY, TRASH }`
- `data class ViewerItem(id, displayName, imageData: Any, isVideo, videoUri: Uri?)`
- Extension mappers: `MediaItem.toViewerItem()` (imageData = uri; videoUri = uri if video) and `TrashItem.toViewerItem()` (imageData priority: localFilePath → localThumbnailPath → uri — matches current `ViewerImagePage` exactly).

## Modified: `ui/PhotoViewerScreen.kt` (the single shared engine)
- New signature adds: `mode: ViewerMode = GALLERY`, `resolveViewerItem: suspend (Long) -> ViewerItem?` (uniform per-page renderer), `onRestore`/`onDeleteForever: (ViewerItem) -> Unit = {}`. `photoIds`→`itemIds`, `initialPhotoId`→`initialItemId`. All defaults preserve the gallery call.
- Pager page resolves `ViewerItem` via `resolveViewerItem(id)` (replaces per-page MediaItem load). All gestures (pinch/transformable, double-tap zoom, pan-bounded, tap-to-toggle UI), `userScrollEnabled = scale == MIN_SCALE`, preload, immersive-mode/insets, animations, `coerceWithinBounds`, constants — UNCHANGED.
- Top bar: shared; add index label `"${currentPage+1}/$size"` beside filename crossfade; hide Info button when `mode == TRASH`.
- Video play overlay: GALLERY only (trash has no playable MediaStore Uri — matches today).
- Bottom bar: `when(mode)` — GALLERY renders existing `ViewerActionBar` (Share/Edit/Favorite/Delete/More) verbatim; TRASH renders new `TrashActionBar` (Restore + Delete Forever) reusing the existing `ActionBarButton` primitive in the same styled container.
- More sheet + Details sheet: built/shown only when `mode == GALLERY`.
- `displayedMedia` (MediaItem) for gallery action callbacks: resolved only in GALLERY (unchanged).
- Delete-Forever confirmation (TRASH only): new `showDeleteForeverDialog` state + `AlertDialog` mirroring existing "Delete All" pattern; on confirm → `onDeleteForever(currentItem)`.

## Modified: `ui/RecentlyDeletedScreen.kt`
- DELETE `TrashViewerDialog` + `ViewerImagePage` (the duplication being removed).
- Restructure to `Box { Scaffold { grid }; if (selected) PhotoViewerScreen(...) }` for true full-screen immersive overlay.
- Wire trash: `mode=TRASH`, `itemIds = items.map{it.id}`, `resolveViewerItem = { id -> items.find{id}?.toViewerItem() }`, `onRestore` calls `viewModel.restoreItem(it, onRestored=onItemRestored)` then clears selection; `onDeleteForever = { vi -> viewModel.deleteForever(vi.id) }`.
- Guard: clear `selectedItemIndex` if it goes >= items.size after list shrinks.
- Restore keeps `onItemRestored` (gallery-refresh hook from prior task). Delete-Forever omits it (item already absent from MediaStore).

## Modified: `ui/TrashViewModel.kt`
- Add `fun deleteForever(itemId: Long)` → `trashRepository.deleteTrashRecords(listOf(itemId))` on IO. Reuses existing batch path with row-id (consistent with `deleteAll()`). `trashItems` Flow auto-refreshes grid; no new effect needed (in-viewer AlertDialog already confirmed intent).

## Modified: `ui/GalleryNavHost.kt`
- Update gallery call to new signature with `mode=GALLERY` and `resolveViewerItem = { id -> viewModel.getMediaItem(id)?.toViewerItem() }`. All existing gallery params/callbacks preserved — 5-button bar, sheets, preload, favorites, intents all unchanged for GALLERY.

## Why each requirement is met
- No duplicate viewer logic: engine exists once; branched only by ViewerMode.
- All gestures preserved: engine code untouched.
- Filename + index (3/20): shared top bar; index added for both modes.
- Trash = Restore + Delete Forever only: Edit/Share/Wallpaper/More/Details gated to GALLERY.
- Modular via ViewerMode (the requested discriminator).
- Gallery untouched: mode=GALLERY keeps existing 5-button bar and sheets verbatim.

## Trade-offs
- Gallery resolves each page twice (ViewerItem render + MediaItem for actions) — same shape as today, not a regression.
- Trash videos show local frame only (no play button) — identical to current trash viewer.
- Delete-Forever feedback is the item leaving the grid (confirm dialog covered intent); no extra snackbar to keep scope tight.
